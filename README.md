# Task Manager API

This is a small Express REST API for managing tasks. Data is stored in memory (so it resets when the server restarts).

## What a task looks like

```json
{
  "id": 2,
  "title": "Create a new project",
  "description": "Create a new project using Magic",
  "completed": false,
  "priority": "low",
  "createdAt": "2026-05-10T09:30:47.911Z"
}
```

Notes:
- `id` and `createdAt` are generated by the server
- `title` and `description` must be non-empty
- `completed` must be a boolean
- `priority` is one of `low | medium | high` (if you don’t send it, it defaults)

## Setup

You need Node.js 18+.

```bash
npm install
```

Start the server:

```bash
node app.js
```

It runs on `http://localhost:3000`.

## Endpoints

Base URL: `http://localhost:3000`

### 1) Get all tasks

`GET /tasks`

- Returns an array of tasks
- Sorted by `createdAt` (newest first)
- You can filter by completion:
  - `GET /tasks?completed=true`
  - `GET /tasks?completed=false`

```bash
curl -s http://localhost:3000/tasks
curl -s 'http://localhost:3000/tasks?completed=true'
curl -s 'http://localhost:3000/tasks?completed=false'
```

If you pass an invalid filter (example below), it returns `400`:

```bash
curl -i 'http://localhost:3000/tasks?completed=yes'
```

### 2) Get a task by id

`GET /tasks/:id`

```bash
curl -s http://localhost:3000/tasks/1
curl -i http://localhost:3000/tasks/999
```

Returns `404` if the id does not exist.

### 3) Create a task

`POST /tasks`

Required body fields:
- `title` (non-empty string)
- `description` (non-empty string)
- `completed` (boolean)

Optional:
- `priority` (`low | medium | high`)

```bash
curl -i -X POST http://localhost:3000/tasks \
  -H 'Content-Type: application/json' \
  -d '{
    "title": "New Task",
    "description": "New Task Description",
    "completed": false,
    "priority": "high"
  }'
```

Invalid payloads return `400`.

### 4) Update a task

`PUT /tasks/:id`

Same validation rules as create.

```bash
curl -i -X PUT http://localhost:3000/tasks/1 \
  -H 'Content-Type: application/json' \
  -d '{
    "title": "Updated Task",
    "description": "Updated Task Description",
    "completed": true,
    "priority": "medium"
  }'
```

- Returns `404` if the id does not exist
- Returns `400` if the body is invalid

### 5) Delete a task

`DELETE /tasks/:id`

```bash
curl -i -X DELETE http://localhost:3000/tasks/1
curl -i -X DELETE http://localhost:3000/tasks/999
```

Returns `404` if the id does not exist.

### 6) Get tasks by priority

`GET /tasks/priority/:level`

`:level` must be `low`, `medium`, or `high`.

```bash
curl -s http://localhost:3000/tasks/priority/low
curl -s http://localhost:3000/tasks/priority/medium
curl -s http://localhost:3000/tasks/priority/high
curl -i http://localhost:3000/tasks/priority/urgent
```

## Testing (Postman)

1. Start the server (`node app.js`).
2. Use the URLs above.
3. For `POST` / `PUT`: set **Body → raw → JSON** and add `Content-Type: application/json`.

## Running the test file

```bash
node test/server.test.js
```
